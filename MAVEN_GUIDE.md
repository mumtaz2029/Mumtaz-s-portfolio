# Maven Project Setup Guide

Your static portfolio website has been converted to a Maven project. This document explains the structure and how to use it.

## New Directory Structure

```
portfolio-website/
├── pom.xml                          # Maven configuration file
├── src/
│   └── main/
│       └── webapp/                  # Web root (served by web server)
│           ├── index.html           # Move main HTML files here
│           ├── 404.html
│           ├── assets/
│           │   ├── css/            # CSS stylesheets
│           │   ├── images/         # Images
│           │   └── js/             # JavaScript files
│           ├── experience/         # Experience section
│           └── projects/           # Projects section
├── target/                          # Build output (generated)
└── README.md
```

## Maven Build Commands

### Clean the project
```bash
mvn clean
```

### Build the WAR file
```bash
mvn package
```

### Clean and build
```bash
mvn clean package
```

### View project information
```bash
mvn site
```

## Project Configuration

**pom.xml Details:**
- **groupId**: `com.mumtazjahan` - Organization identifier
- **artifactId**: `portfolio-website` - Project name
- **version**: `1.0.0` - Project version
- **packaging**: `war` - Web Archive format (standard for web projects)

## Next Steps

1. **Move files to Maven structure**: Copy your HTML, CSS, JS files to `src/main/webapp/`
   - Move `index.html` → `src/main/webapp/index.html`
   - Move `404.html` → `src/main/webapp/404.html`
   - Move `assets/` → `src/main/webapp/assets/`
   - Move `experience/` content → `src/main/webapp/experience/`
   - Move `projects/` content → `src/main/webapp/projects/`

2. **Add resources** (optional): Place any configuration files in `src/main/resources/`

3. **Build the project**:
   ```bash
   mvn clean package
   ```
   This creates a WAR file in `target/portfolio-website.war`

4. **Deployment options**:
   - Deploy to Apache Tomcat: Copy the WAR file to `tomcat/webapps/`
   - Deploy to Docker: Use the WAR file with a Java web server container
   - Keep using Netlify: No changes needed for Netlify deployment

## Benefits of Maven for Static Sites

✓ **Build Automation**: Package static assets with other resources
✓ **Consistency**: Standard project structure across teams
✓ **Version Control**: Easy version management and releases
✓ **Docker Ready**: Can easily containerize the application
✓ **CI/CD Integration**: Works seamlessly with Jenkins, GitHub Actions, etc.
✓ **Documentation**: Generate project documentation with `mvn site`

## Notes

- The `target/` directory is auto-generated by Maven and should not be committed to Git (added to `.gitignore`)
- Your Netlify deployment workflow remains unchanged
- You can continue serving files locally using any static server
